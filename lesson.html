.NET Core

Уроки .NET Core

Платформа ASP.NET Core собою представляє технологію від компанії Microsoft, призначену для створення різного роду веб-додатків: від невеликих веб-сайтів до великих веб-порталів і веб-сервісів.

З одного боку, ASP.NET Core є продовженням розвитку платформи ASP.NET. Але з іншого боку, це не просто черговий реліз. Вихід ASP.NET Core фактично означає революцію всієї платформи, її якісна зміна.

ASP.NET Core є кросплатформенним opensource-проектом. Весь код проекту доступний на GitHub, і тому кожен може бачити реалізацію всього функціоналу цієї платформи і робити до неї зміни, якщо це буде потрібно.

Додаток ASP.NET Core може працювати з двома виконуваними середовищами: .NET Core і з повною версією фреймворка .NET.

.NET Core представляє модульну крос-платформену виконуючу середу, яка спрощує розгортання програми. .NET Core програми можна запускати не тільки на ОС Windows, але і на Linux і MacOS.

Завдяки модульності всі необхідні компоненти веб-додатки можуть завантажуватися як окремі модулі через пакетний менеджер Nuget.

ASP.NET Core - більш оптимізована для використання в хмарних сервісах і має вбудовану підтримку впровадження залежностей.

Перша консольна програма

Створюємо папку із проектом і заходимо в її. 
mkdir helloworld
cd helloworld
Виконаємо команду, що створює новий проект. 
dotnet new
Заходимо із допомогою свого середовища розробки у папку проекту. Моїм середовищем розробки є Visual Studio Code, і тому я використовую наступну команду. 
code .
У кожного .NET Core проекта є файл project.json. У ньому визначаються всі залежності, що використовуються серверною стороною проекту, а також налаштована конфігурація. Переініціалізувати проект, в залежності від вмісту цього файлу, можна із допомогою команди 
dotnet restore
Введемо цю команду для ініціалізації.
Спробуємо запустити проект. 
dotnet run
Після виконання команди в консолі виводиться надпис "Hello world". Відкриємо файл Program.cs. Бачимо код програми. 
  1. using System;
  2.  
  3. namespace ConsoleApplication
  4. {
  5.     public class Program
  6.     {
  7.         public static void Main(string[] args)
  8.         {
  9.             Console.WriteLine("Hello World!");
  10.         }
  11.     }
  12. }
Напишемо щось своє, наприклад, програму, що групує цілі числа по парності. 
  1. using System;
  2. using System.Linq;
  3.  
  4. namespace ConsoleApplication
  5. {
  6.     public class Program
  7.     {
  8.         public static void Main(string[] args)
  9.         {
  10.             var array = new Int32[] { 1, 2, 3, 4, 5, 6, 7, 8 };
  11.  
  12.             var result = 
  13.                 from x in array
  14.                 group x by x % 2 == 0;
  15.  
  16.             foreach (var numbersGroup in result)
  17.             {
  18.                 Console.WriteLine(numbersGroup.Key ? "Парні" : "Непарні");
  19.                 foreach (var x in numbersGroup)
  20.                 {
  21.                     Console.WriteLine(x);
  22.                 }
  23.             }
  24.         }
  25.     }
  26. }
  27.  
Для запуску програми в консолі введемо, аналогічно попередній програмі, наступну команду. 
dotnet run

Створюємо сайт із ASP.NET Core

Створимо новий проект. 
mkdir aspnetcoreapp
cd aspnetcoreapp
dotnet new
Додамо в project.json ще одну залежність, а саме Kestrel HTTP.

Переініціалізуємо проект.

dotnet restore

Створимо файл Startup.cs, де створимо клас Startup, що визначає логіку обробки запитів. У цьому класі зробимо так, щоб при завантаженні сторінки виводився серверний час. 

  1. using System;
  2. using Microsoft.AspNetCore.Builder;
  3. using Microsoft.AspNetCore.Hosting;
  4. using Microsoft.AspNetCore.Http;
  5.  
  6. namespace aspnetcoreapp
  7. {
  8.     public class Startup
  9.     {
  10.         public void Configure(IApplicationBuilder app)
  11.         {
  12.             app.Run(context =>
  13.             {
  14.                 return context.Response.WriteAsync(
  15.                     "Time now: " + DateTime.Now.ToString()
  16.                 );
  17.             });
  18.         }
  19.     }
  20. }

Перепишемо код в Program.cs, щоб написати код, що запустить сайт на локальному сервері.

  1. using System;
  2. using Microsoft.AspNetCore.Hosting;
  3.  
  4. namespace aspnetcoreapp
  5. {
  6.     public class Program
  7.     {
  8.         public static void Main(string[] args)
  9.         {
  10.             var host = new WebHostBuilder()
  11.                 .UseKestrel()
  12.                 .UseStartup<Startup>()
  13.                 .Build();
  14.  
  15.             host.Run();
  16.         }
  17.     }
  18. }

Тепер після запуску проекту ми можемо зайти на відповідну адресу на локальному сервері, тим самим завантаживши сторінку із написаним на ній серверним часом.
